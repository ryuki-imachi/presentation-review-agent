# S3 ライフサイクル設定ガイド

アップロード済み音声ファイルや文字起こし結果を一定期間後に自動削除したい場合、S3 ライフサイクルルールを設定します。

## 設定手順（AWS コンソール）

1. AWS コンソール → S3 → 該当バケットを開く
2. 「管理」タブ → 「ライフサイクルルールを作成」をクリック
3. 以下の設定で作成:

| 項目 | 設定値 |
|---|---|
| ルール名 | `auto-delete-private-files` |
| プレフィックスフィルター | `private/` |
| ルールアクション | 「オブジェクトの現行バージョンを有効期限切れにする」 |
| 有効期限切れまでの日数 | `30`（任意で変更） |

## 注意事項

- ライフサイクルルールは全ユーザーの `private/` 配下に一律で適用されます
- 個人利用の場合は 30 日で十分ですが、用途に応じて調整してください
- 文字起こしのみ長く保持したい場合は、プレフィックスを `private/*/audio/` に絞る方法もあります
- ルール設定後、実際の削除はS3のバックグラウンド処理で行われるため、即時削除ではありません

## 手動削除

アプリの分析結果画面にある「データを削除」ボタンで、音声ファイルと文字起こし結果を即座に削除できます。
